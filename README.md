# CHRONO//NOIR

> // DISPATCH PROTOCOL ONLINE

A themed ChatGPT clone built with Laravel, Inertia, and Vue 3. The application features **DISPATCH**, a cold, militaristic temporal crisis dispatcher aimed at users interested in time management and productivity, wrapped in a cyberpunk aesthetic.

Built for the Development Project course (SGBD) — IDV4, IFOSUP Wavre.

---

## Tech stack

- **Backend**: Laravel 12 (service-based architecture)
- **Frontend**: Vue 3 (Composition API), Inertia.js
- **Styling**: Tailwind CSS v4, shadcn-vue
- **Database**: SQLite
- **LLM API**: OpenRouter (GPT, Gemini, Claude, Grok…)
- **Streaming**: Server-Sent Events via `@laravel/stream-vue`
- **Server**: Nginx + PHP-FPM

---

## Features

- **Model selector** — switch LLMs on the fly, preference stored per user
- **Conversation history** — automatic saving, title generated by the AI on the first message
- **Real-time streaming** — responses rendered token by token via SSE
- **Custom instructions** — configure DISPATCH's behavior (identity / tone / directives) through the Protocols page
- **Delete with confirmation** — custom modal, cascading deletion of messages
- **Generation resume** — a conversation left without a reply automatically restarts the stream on reload

## Installation

```bash
# Clone the repository
git clone <repo-url>
cd MiniChat

# PHP dependencies
composer install

# JS dependencies
npm install

# Configuration
cp .env.example .env
php artisan key:generate

# Database
touch database/database.sqlite
php artisan migrate
```

### Environment configuration

In the `.env` file, set your OpenRouter key:

```env
OPENROUTER_API_KEY=your_key_here
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
```

---

## Running the app

Start the system services:

```bash
sudo systemctl start nginx php-fpm
```

Start the frontend dev server:

```bash
npm run dev
```

> **Note**: SSE streaming requires `fastcgi_buffering off` in the Nginx configuration. Apache does not release chunks incrementally and breaks real-time streaming.

## Architecture

### Data model

| Table | Role | Notes |
|-------|------|-------|
| `users` | Users + custom instructions | JSON columns `identity`, `tone`, `commands` + `favorite_ai` (string) |
| `chats` | Conversations | **UUID** primary key |
| `messages` | Individual messages | `role` enum field (`user` / `LLM`) |

**Relations**: `User` 1—N `Chat` 1—N `Message` (composition, cascading deletion).

### Service layer

- `ChatService` — shared logic (message saving, history building, title generation)
- `SimpleAskService` — standard OpenRouter calls (title)
- `SimpleAskStreamService` — SSE streaming (chunk parsing, accumulation, flush)

### Streaming request lifecycle

1. User sends a message → `ask()` saves it and redirects
2. `show()` computes `pending = true` (last message is from the user)
3. A Vue `watch` detects `pending` and calls `/stream`
4. `SimpleAskStreamService` streams tokens via SSE
5. On completion, the response is persisted then the title is generated
